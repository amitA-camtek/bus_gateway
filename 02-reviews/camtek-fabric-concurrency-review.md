## 1. CRITICAL findings & resolutions

| # | Finding (reviewer) | Resolution decided |
|---|---|---|
| CC1 | **Class-A journal flush on the caller thread falsifies the ≤1 ms bound** — `FlushFileBuffers` spikes 10–100 ms under normal tool-disk load; the scan thread stalls on disk — the exact ToolApiPublisher disease the design cures (bus F1) | **Single journal-writer thread + group commit** (one flush per batch/N ms). Caller = lock-free enqueue only (true ≤1 ms, no disk). Class-A contract restated honestly: *durable against process crash immediately (OS page cache survives process death); durable against power loss within the flush interval ≤X ms* — acceptable because wafer results also exist at their stable path. Disk-slow fault injection added to TestKit |
| CC2 | **Two threads mutating one journal file = corruption/silent loss** — caller appends while pump "deletes"; compaction-rename loses concurrent appends (bus F6) | Same mechanism: **only the journal thread touches journal files**; "delete" = **append an ack-tombstone** `{"ack": seq}` (O(1), crash-safe); compaction = tmp + `ReplaceFile` with no racing appender by construction. The `FailedMessagesHandler` precedent claim dropped (false analogy — that was single-threaded) |
| CC3 | **Multi-subscriber class-A E2E-ack undefined** — ack-on-first = silent loss; ack-on-all without lifecycle = journal pinned forever + tracking-map leak (bus F3b) | Normative semantics: **E2E_ACK per (message, expected-subscriber-set snapshotted at PUB)**; a subscriber that disconnects/unsubscribes is removed from all pending sets (its durability claim ends with registration); zero-subscriber rule = degenerate case; publisher disconnect purges its routing entries; tracking map bounded by Σ queue capacities, contract-tested |
| CC4 | **Single pump thread: duplex write-write deadlock + head-of-line blocking of Ttl'd commands behind own bulk traffic** (bus F2) | **Reader/writer split (overlapped I/O) on both client and broker** — a peer must always drain reads regardless of write progress. **Priority lanes** in send queues and per-connection broker writers: `REQ/REPLY` > A > B > C (weighted, no total starvation); per-frame write deadlines with reconnect-on-stall |
| CC5 | **Gateway acks class-A before durable ownership** — DELIVER_ACK → journal deleted → message exists only in a RAM channel → native TsmcShim crash loses it; "zero silent loss" false as drawn (E2E-1) | **BusSource sends DELIVER_ACK only after a write-ahead spool append** — the spool becomes a WAL, not a failure bin. Fault case "gateway crash between DELIVER_ACK and sink publish" added to TestKit assertion 5 |
| CC6 | **NACK has no retry schedule — healthy-looking livelock** (hung BusSource + green HTTP health + pipe never drops = stranded journal forever) (E2E-2) | Normative NACK redelivery: per-topic **exponential backoff + jitter, journal `seq` order, one bounded in-flight window**, independent of reconnect; broker **`RESUME`** frame (queue below low-watermark) so publishers don't blind-poll; **gateway health = BusSource consumption liveness** (last-DELIVER-processed token), ToolHost compares broker-delivered vs gateway-processed high-waters |
| CC7 | **GEM process has no degraded contract** — host ONLINE/REMOTE while `gui.commands` undeliverable; fab discovers the failure via HCACK storms (E2E-3, connectivity M5) | **GEM-shim degraded-mode table added (fabric Part III)**: bus unreachable ≥T → host-visible control state degrades (ONLINE-LOCAL / dedicated alarm per E30 semantics) and REMOTE grant refused until bus heartbeat green; bus-dark command → *deliberate* HCACK denial code, never a timeout; host-link state published to `tool.telemetry`; the 4 HSMS×bus states enumerated; "HSMS up / bus down" = GEM record-replay test scenario |
| CC8 | **Broker quarantine = permanent fabric blackout** — ToolHost's leaf-child policy applied to the dependency of everything; `EnsureBusRunning` can't cure it (service is up) (E2E-4) | Per-child quarantine class: **broker (and gateway) `quarantine: never`** — infinite restarts at max backoff + escalating alarm; `EnsureBusRunning` handles "service up / child stopped" via `POST /children/broker/restart`; broker counters **pushed** to ToolHost on heartbeat so the last snapshot survives broker death; broker children-config entry added (startOrder 0, pipe-probe type = new ToolHost work item, priorityClass AboveNormal) |
| CC9 | **BusAdapter marshal underspecified → four-party deadlock** (UI blocked in COM → ToolManager CB → GEM waits reply → BusAdapter parked in `Invoke` → UI). Ttl gate ran pre-marshal; NACK never sent. Mirror hazard: legacy `DoEvents` pump executes commands reentrantly mid-transition (AOI F1) | Normative marshal rules (AOI §3.5.4): **blocking `Invoke` banned; `BeginInvoke`-post only; ACCEPTED = "queued to the UI dispatcher"** (reply sent on successful post); **Ttl re-checked at dequeue** on the executing thread — expired-at-dequeue → command-expired event + compensation; **requester-side deadline mandatory** in the bus contract; **command serialization gate** (one `gui.commands` in flight; deferred/NACKed during state transitions) |
| CC10 | **P4 NACK has no COM channel** — customer holds S_OK waiting for a done-event that never fires; plus a 3-process Invoke cycle through the frozen FalconWrapper fan-out (AOI F2) | **Per-command compensation table** (P4 deliverable): Ttl-expiry/NACK ⇒ synthesized completion/failure `Fire*` (`ManualScanDone`, export-failed, defined return values for value-returning callbacks); dispatch = post + return immediately (today's `async void` semantics, made explicit); bare `Control.Invoke` on COM inbound threads banned |
| CC11 | **RobotUI absorption dual-STA deadlock** — module spawns its own STA pump + DoEvents busy-loop + per-dialog STAs; mutual-Invoke across two STAs is permanent (AOI F3) | **A-4 audit replaced by a structural acceptance criterion: single-STA consolidation** — `frmRobot` created on AOI's main UI thread, per-dialog STA threads become plain `ShowDialog`, all module COM proxies confined to one apartment. Transitional dual-STA (if unavoidable): written thread-ownership contract, GIT marshaling, **post-only cross-STA calls in both directions** |
| CC12 | **Error-storm amplification** — class-A telemetry upgrades every error line (today: a log write) into fsync+broker+ack+WAN; a flapping sensor = 300k class-A messages, fleet-amplified (load F1) | **Storm control as a topic-contract property of `tool.telemetry(error)`, in the client library**: coalesce by `(source, errorCode)` sliding window — first occurrence immediate, then class-A summaries ("code X × N") every 10 s; token bucket per source (10/s sustained, 100 burst, overflow counted). TestKit: 1 kHz × 60 s ⇒ downstream ≤10 msg/s, journal bounded, publish bound held |
| CC13 | **The spool drain assumed by Flow G1 does not exist in the code** — restore only at process start, 10k cap → unread overflow, `TryWrite` into a 1000-channel re-spools ~9k of 10k; one quiet outage hour = 3–4 manual restarts (load F2) | Wave-0 spool prerequisites grow from two to **four**: (a) poison-loop retry cap/age **split from outage** (below), (b) overflow consumed+alarmed, (c) **periodic drain loop** (every 60 s while spool non-empty and sink healthy; capped ~50 msg/s, oldest-first by seq, interleaved with live traffic), (d) **backpressured restore** (`WriteAsync`, not one-shot `TryWrite`). Acceptance: 1-h backlog drains without restart in <10 min |
| CC14 | **The planned spool "age limit" fix institutionalizes class-A loss during multi-day sink outages** — poison and outage conflated (E2E-1b, connectivity C2) | **Poison vs outage split, normative**: retry-cap/age apply **only to messages failing while the sink reports connected** (rejection classes); sink-unreachable messages retry forever with backoff, bounded by **disk quota + alarm** (spool depth + oldest-age on `tool.telemetry`); quota-full policy defined per subscription type |
| CC15 | **SVC lane converts fail-fast COM errors into 30-s UI freezes** — synchronous UI-thread calls, no deadline/circuit breaker/fallback (connectivity C3) | **Mandatory SVC client policy (AOI Appendix B)**: per-call deadline (2–5 s default), fail-fast circuit breaker on known-down channel, per-service degraded behavior (SystemLogger → local file; JobProvider → error dialog); contract test "host killed mid-call → caller unblocked < deadline" |
| CC16 | **No endpoint source of truth** — Fleet IP hardcoded twice in one file; 8–9 config files per tool; fingerprint doesn't cover endpoint drift; fab re-IP = 100+ hand edits (connectivity C1) | **Tool-level endpoint manifest** (one file, ToolHost-owned; children receive endpoints from it); **endpoint-config hash added to the fleet fingerprint** on `tool.telemetry`; DNS name for Fleet (IP fallback only); immediate cheap fix: deduplicate the Fleet address in today's appsettings |

## 3. The composite-fault theme

Every reviewer converged on one meta-finding: **the failures live in hand-offs under concurrency, while the TestKit tested components in isolation.** The kit therefore gains composite scenarios: ack-vs-spool ordering crash, NACK-with-healthy-pipe, replay-into-cold-consumer, R-R-behind-bulk, fsync-under-co-load, storm, herd, outage-drain (T-L series + assertion-5 extensions).

## 4. Resolution conflicts decided

- Bus-reviewer F5a wanted "reply only after the marshaled delegate commits"; AOI-reviewer F1 and E2E-reviewer 5.3 wanted "reply on successful post." **Decided: reply = ACCEPTED on successful post** (keeps the Ttl budget honest, avoids parking pool threads), **with the final Ttl gate at dequeue** and expired-at-dequeue → command-expired event + compensation (CC10). The residual host-told-failed/tool-runs window is closed by the dequeue gate; the residual host-told-accepted/command-expired window is compensated and documented.

# FINAL VERDICT: **all CRITICAL and MAJOR concurrency/connectivity/load findings RESOLVED and verified**

The designs now carry, normatively: a publish path with an unconditional ≤1 ms caller bound (no caller-thread disk), a corruption-free single-writer journal with an honest two-tier durability contract, defined multi-subscriber ack semantics, deadlock-free split-I/O with priority lanes, a WAL gateway with poison/outage-separated retry and a real drain loop, deadlock-safe UI marshaling with two-stage Ttl gates and compensations, a structural single-STA rule for absorbed modules, degraded-mode contracts for every process incl. the fab-facing GEM door, endpoint configuration with a single source of truth, storm control, herd control, a load model with derived buffer sizes, and a TestKit whose composite-fault scenarios (assertions 5/11/12/13/14) would have caught every finding in this review.

Standing engineering conditions (not document defects): the P0 measurements (group-commit interval under co-load, single-instance ceilings, GEM pre-Ttl hop latencies, TsmcSink service time) and the five live-bug work items (§5) — LB3 (ToolId=0) and LB4 (NonBlockingUITask timeout) are recommended for immediate ADO filing independent of the program.
